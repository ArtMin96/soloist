## 0. What Soloist is

Soloist is a **native Linux (Ubuntu, x86_64) process-supervisor + AI-agent-coordination workspace** ‚Äî a
clean-room, open rebuild of the macOS-only **Solo** (`soloterm.com`, v0.8.2). It runs your dev stack and
CLI coding agents from one dashboard, keeps them alive, and gives agents a shared project-scoped workspace
(logs, status, todos, scratchpads, locks, timers) over **MCP** ‚Äî driven by a committable `solo.yml`.

It's a *metaharness* ‚Äî not a coding agent, not a terminal emulator, not a worktree orchestrator. Scope: `plan/00-vision-and-scope.md`.

**Stack:** Tauri v2 + Rust core + React/TypeScript + xterm.js. **Architecture:** Hexagonal (ports &
adapters); pure framework-free domain core with 8 bounded contexts (C1‚ÄìC8); actor-model supervision;
event-driven + CQRS-lite; SQLite for durable state. Full design: `plan/04-‚Ä¶` (mandatory reading, ¬ß3).

## 3. Load-bearing facts (invariants ‚Äî treat as constants)

### Confirmed decisions (locked)
- **D1 Stack:** Tauri v2 + Rust + React/TS + xterm.js. **D2 Target:** Ubuntu 20.04+, **x86_64 only** (`.deb` + `.AppImage`; no arm64/macOS/Windows). **D3 Licensing:** dropped entirely. **D4 MCP:** `soloist-mcp` binary, **stdio** transport. **D5 `solo.yml`:** byte-compatible with Solo's schema. **D6 Storage:** SQLite durable, in-memory runtime.
- **Coordination layer (Phase 9, C6) is v1 scope** ‚Äî matrix rows G1‚ÄìG11 + E7.
- **Auto-summarization defaults OFF** ‚Äî opt-in only. Core must never hard-depend on an LLM. Heuristic idle detection is always on.
- **Git:** private GitHub remote. Commit per phase. `PROGRESS.md` is the human-readable ledger; git history complements it, not replaces it.

### The real `solo.yml` schema (D5 ‚Äî keep byte-compatible)
```yaml
name: storefront                 # optional
icon: assets/project-icon.png    # optional
processes:                       # MAP keyed by process name (NOT a list)
  Web:
    command: npm run dev         # required
    working_dir: null            # optional
    auto_start: true             # our default true (Solo's default is a documented gap)
    auto_restart: false          # default false
    restart_when_changed: []     # glob list (file-watch restart)
    env: {}                      # per-process env (highest precedence)
```
1 MB limit. Trust-gated. Synced via hash-diff + debounce. Never silently rewrite the user's `solo.yml`; app-local additions go in app state (Visibility::Local vs Shared).

### Closed domain enums (exhaustive `match`; never stringly-typed)
```rust
enum ProcessKind { Command, Agent, Terminal }
enum ProcStatus  { Stopped, Starting, Running, Crashed, Restarting, Stopping, RestartExhausted }
enum AgentActivity { Idle, Permission, Thinking, Working, Error }   // the 5-state idle FSM
enum Trust       { Untrusted, Trusted { variant_hash: Hash } }
enum Visibility  { Shared, Local }
```

### Hard behavioral invariants
- Crash auto-restart capped at **10 restarts / 60 s ‚Üí `RestartExhausted`**.
- Log ring buffer **5,000 lines**; raw scrollback cap **256 KB**; oldest dropped; global cap across all processes. **No buffer/channel/retry without a ceiling.**
- Graceful stop: SIGTERM ‚Üí ~5 s grace ‚Üí SIGKILL, then **reap** (no zombies/orphans). Spawn into a fresh process group; kill the group, never a bare PID.
- File-watch restarts: **debounced**, **trusted-only**.
- HTTP API: loopback `127.0.0.1:24678`, `X-Soloist-Local-Auth` header on mutations, CORS localhost-only.
- MCP identity: `SOLOIST_PROCESS_ID` env + `bind_session_process`/`register_agent`/`whoami`; tools honor the **trust gate** and **effective project scope** (a tool cannot touch another project).
- Data dir: `SOLOIST_APP_DATA_DIR` override; default XDG `~/.local/share/soloist/`.

### Non-negotiable architectural rule
**`crates/core` imports NO app frameworks** ‚Äî no `tauri`, `rmcp`, `axum`, `rusqlite`. CI enforces this. Everything OS/UI/transport/storage is an adapter behind a port. If you feel the urge to `use tauri` in core, add a port instead.

## 6. Performance, size & responsiveness budget (first-class gates ‚Äî measured, never fabricated)

- **Bundle size:** target low tens of MB. Measure real `.deb`/`.AppImage` in Phase 12; record in `PROGRESS.md`. Lazy-load heavy deps (mermaid, WebGL addon); code-split; tree-shake. Add a dep only when it clearly pays for itself.
- **Runtime footprint:** idle RSS < ~150 MB. Phase 13 measures; if it misses, document the gap + plan ‚Äî never guess a number.
- **Responsiveness:** ~60 fps under chatty processes. Coalesce terminal output per animation frame; virtualize long lists/scrollback; never block the main thread or `tokio` runtime. Backpressure (¬ß8) protects this.
- **Build:** LTO + `codegen-units = 1` + stripped symbols in release. No debug bloat.
- **Optimize measured bottlenecks only.** Prefer cheaper algorithms/data structures; avoid needless clones in hot paths (PTY read loop, event fan-out); pool SQLite connections; sample metrics on an interval. No speculative micro-optimization without profiling evidence.

**Performance pass workflow (MANDATORY when optimizing speed/size/CPU/memory/responsiveness):**
1. **Skills + valid sources first.** Invoke matching Tauri skills (`tauri-binary-size`, `tauri-calling-frontend`, `tauri-ipc`, `tauri-process-model`, `tauri-linux-packaging`) and confirm every API/flag/config against official docs or `context7`. No assumption, no fabrication.
2. **Measure before and after.** Profile the proven hot spot: `just bloat`, `just bundle-size`, the nightly soak (¬ß8) for RSS/FD/task drift, webview devtools for repaints. A change with no before/after number is speculative and is rejected. Record numbers in `PROGRESS.md`.
3. **Stay inside the architecture and budget.** Performance lives in adapters and the composition root, almost never in pure `core`. Keep hexagonal layering, bounded caps, and backpressure intact. Correctness and clarity outrank speed.
4. **Locked non-changes ‚Äî do NOT touch:** `panic = "unwind"` (supervisor catches task panics; `abort` breaks fault isolation), `freezePrototype = false` (breaks xterm.js), `Cargo.lock` brotli pins, release `opt-level` (packaging-phase measured decision), `removeUnusedCommands` (only safe when ACL is complete + runtime verified). When unsure if a lever is locked, ask (¬ß12).

## 8. Engineering rules (anti-rot contract ‚Äî rigid)

Full rationale in `plan/04`. These are rigid ‚Äî adapt the *feature*, never these:

- **Hexagonal:** core is pure; OS/UI/MCP/HTTP/CLI/SQLite/PTY are adapters behind ports. Core is the only source of truth; adapters hold no business state; React renders a pushed read-model projection and holds **no business logic**.
- **One behavior, many frontends:** Tauri UI, MCP, and HTTP/CLI all route to the **same** core command. Never reimplement an action (e.g. "restart") per adapter.
- **Actors, not shared mutable state:** each managed process is one supervised `tokio` task that solely owns its child/PTY/stdin/exit-watcher. Interact via bounded `mpsc` + emitted events. No big `Mutex` over domain state. Single-writer per aggregate.
- **FSMs are contracts:** state changes are explicit functions returning `Result<NewState, IllegalTransition>`. No ad-hoc field mutation.
- **Errors are values:** typed errors (`thiserror`) at boundaries; **no `unwrap()`/`expect()`/`panic!` in long-running tasks** (clippy-denied in `core`). A dying child, bad `solo.yml`, full disk, missing agent binary ‚Äî all expected and handled.
- **Fault isolation & self-supervision:** each actor/sampler runs under a supervisor that catches panics, marks the unit `Error`, and keeps the app alive. Internal tasks (metrics, file-watch, event pump) are themselves supervised and auto-restart with backoff.
- **Bounded everything + backpressure:** caps on buffers/channels/retries; coalesce chatty output per frame; debounce file events; rate-limit restarts (10/60 s) and summaries. Reclaim OS resources deterministically (close PTYs/FDs in Drop/cancel; reap process groups). A start/stop loop of N processes ends at the same PID/FD/task count.
- **Graceful degradation:** no optional subsystem (summarizer, port discovery, notifications) can crash the core.
- **Deterministic shutdown:** stop accepting commands ‚Üí cancel watchers/timers/samplers ‚Üí `stop_all()` (reap) ‚Üí flush SQLite ‚Üí exit. No orphans on quit.
- **Persistence split by lifetime:** ephemeral (registry/PIDs/metrics/PTY buffers) in memory; durable (trust, projects, settings, todos, scratchpads, kv, locks/leases) in SQLite (WAL, transactions, versioned migrations). Optimistic concurrency (revision guards) for scratchpads/todos. Leases carry TTL + owner `ProcessId`, auto-release on expiry/owner-close.
- **Security in the core:** trust gate enforced in core (not UI) for `start*`/`restart*`/auto-*, per (project, command-variant hash). MCP/HTTP honor scope + auth.
- **Comment & naming discipline:** doc comments on public items; the rare comment explaining a *non-obvious* decision ‚Äî nothing else. **Never** write phase numbers, R-phase tags (e.g. `R1`, `Phase 5`), plan/doc citations (e.g. `plan/04 ¬ß6`), changelog narration, `placeholder`/`TBD` notes, or comments that restate the code. **The same prohibition applies to all names** (files, modules, functions, test names, identifiers) ‚Äî no `r1_reach.rs`, no `phase5_test`. Name things for *what they are*, permanently. Temp notes/files added mid-phase are removed before the phase ends.

## 15. Codebase discipline (clean, reusable, single-source ‚Äî MANDATORY)

Hard rules, checked every phase. A change that violates one is **not done**:

- **Single source of truth.** Every concept defined once and referenced everywhere else. Rust domain enums in `core`; TS mirror in **one** `domain.ts` and nowhere else. Cross-boundary constants are a single named `const` per side.
- **No magic strings or numbers.** Never compare against or emit a bare status/kind string or a bare numeric limit/timeout. Use the enum or a named `const`. Applies in adapters and frontend too.
- **DRY ‚Äî one place to change.** A requirement changes in exactly one place. Editing "the same thing" in two files is a signal to refactor, not to make two edits.
- **Small, single-purpose files.** A file does one thing; ~400-line non-test source is a split smell to act on. Many small focused modules over one large one, in Rust and the frontend.
- **Clear domain separation.** Logic in its context; adapters thin; React renders projections. Don't smear one concern across layers.
- **Component-based frontend.** Structure: `domain.ts` (types) ¬∑ `api.ts` (typed IPC only) ¬∑ `store/` (pure reducers + hooks) ¬∑ `components/` (small, reusable, presentational). No business logic in components; no huge `App.tsx`; no duplicated markup ‚Äî extract a component.
- **Tests test behaviour.** Every test exercises real logic and can fail for a real reason. No placeholder/empty tests, no tautological asserts. Never `#[ignore]`, skip, or comment out a test to dodge a red ‚Äî fix the cause or report it red (¬ß12). If a module has nothing meaningful to test yet, it has no test yet ‚Äî that's honest. Delete pretend tests on sight.
- **No unnecessary code or comments.** Doc comments on public items; the rare non-obvious-decision comment only. No dead code, no speculative abstraction (YAGNI).
- **Built to change safely.** Typed boundaries, exhaustive `match`, ports over concretions, names that say what they mean. Keep tunables (limits, timeouts, paths, default policies) as named values in one place so they can be promoted to config without touching call sites ‚Äî don't pre-build config you don't need yet.

## 16. Architecture & structure rules ‚Äî how to build any change (MANDATORY)

Detailed blueprint: **`plan/06-codebase-blueprint-and-cleanup.md`**. These are the load-bearing invariants ‚Äî **do not diverge**; if you think a change needs to break one, stop and surface it (¬ß12):

- **Behavior ‚Üí context ‚Üí port ‚Üí one fa√ßade.** All business logic in a bounded context in `crates/core`, behind ports (traits in `core::ports`), exposed through the **single `Facade` (C8)**. Adapters and React hold no business logic. Never reimplement an action per adapter ‚Äî route to core. Never add a domain `if` to an adapter.
- **Adapters are independent crates; dependency points one way.** Each surface (Tauri UI, MCP, HTTP, CLI) is its own crate depending only on `core`/`ipc`. `core` depends on nothing app-specific (CI-enforced). Removing an adapter = drop the crate from the workspace + the composition root; nothing else references it.
- **Optional subsystems are ports with `Noop` defaults.** Core always holds a port and never branches on "is it present?". Add new optional subsystems the same way (`plan/06` ¬ß5.2).
- **One composition root per binary.** `crates/app/src/lib.rs::build_facade` is the **only** place real-vs-`Noop` adapters are chosen; it assembles `core::ports::CorePorts` via its builder and hands it to `Facade::new`. No other code constructs adapters. A future port is one field on `CorePorts`. Tests are alternate composition roots using `core::testing` fakes.
- **Single source of truth, everywhere.** Rust enum in `core`; TS mirror in one `domain.ts`; one command/event-name constant per side. Shared test fakes in `core::testing`, never re-rolled per crate.
- **Small files; tests in separate files, honest.** Split non-test source at the ~400-line smell. New tests live in their own file: unit tests of private items via `#[cfg(test)] #[path = "x_tests.rs"] mod tests;` (stays a child module, so it still reaches private items); adapter integration tests in `tests/`. Inline only when there is no other way. Every test must exercise real behavior.
- **Reach for a pattern when its trigger fires, not before.** `plan/06` ¬ß4 table: FSM for legal state transitions; Registry for a growing handler set (MCP tools, agent providers ‚Äî never a giant `match`); Strategy for per-provider behavior; Repository per durable aggregate; Builder when >4 constructor collaborators. No speculative abstraction (YAGNI).
- **Use the recipes.** Adding a context behavior, port+adapter, MCP tool, HTTP/CLI/Tauri command, `DomainEvent`, or UI surface each has a checklist in `plan/06` ¬ß5. Follow it so the change lands in the right layer with dependency rule, single-source, and DRY intact.
